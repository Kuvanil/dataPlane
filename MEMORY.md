## Known constraints / gotchas

- `SECRET_KEY` (JWT) must be overridden via env var in production ‚Äî code default is dev-only.
- No Alembic-equivalent migrations ‚Äî destructive schema changes (drop/rename column) need a human sign-off, not an auto-implementation.
- Celery beat requires its own `beat` service entry in `docker-compose.yml` ‚Äî periodic tasks silently won't fire without it.
- `frontend/src/lib/api.ts` is the single chokepoint for HTTP calls and 401 handling (`setUnauthorizedHandler`) ‚Äî don't add a parallel fetch or auth-redirect path.
